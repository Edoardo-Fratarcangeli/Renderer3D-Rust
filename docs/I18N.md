# Internationalization (in-app UI)

The application interface is multilingual. This is **separate** from the
installer localization (covered in [PACKAGING.md](PACKAGING.md)).

## Languages

English (`en`, fallback), Italian (`it`), Spanish (`es`), French (`fr`),
German (`de`).

## How it works

- Translations live in `locales/app.yml` and are embedded at compile time by
  `rust_i18n::i18n!("locales", fallback = "en")` in `src/lib.rs`. There is no
  runtime file to ship.
- `src/i18n.rs` picks the startup language as: **saved choice → OS language →
  English**. The OS language is detected with `sys-locale`.
- The choice is persisted per user under the OS config directory
  (`<config>/rendering3d/language`, via the `dirs` crate).
- The user changes language live from **Settings → Language**; the UI updates on
  the next frame (egui redraws continuously).

## Using a translation in code

The `t!` macro is available crate-wide (`#[macro_use] extern crate rust_i18n;`).
egui widgets want a `String`/`&str`, so call `.to_string()`:

```rust
ui.heading(t!("settings.global_heading").to_string());
ui.button(t!("common.add").to_string());
ui.label(format!("{}:", t!("settings.language")));   // Display works directly
```

## Adding a new string

1. Add the key to `locales/app.yml` with a value for **every** language:
   ```yaml
   "panel.my_label":
     en: My label
     it: La mia etichetta
     es: Mi etiqueta
     fr: Mon étiquette
     de: Meine Beschriftung
   ```
2. Use it: `ui.label(t!("panel.my_label").to_string());`

A missing key in a non-English language falls back to English automatically.

## Adding a new language

1. Add its code to every key in `locales/app.yml`.
2. Add `(code, "Native Name")` to `LANGUAGES` in `src/i18n.rs`.

## Translation files

`rust-i18n` loads and **merges every file** under `locales/`, so translations are
split per area for maintainability:

- `locales/app.yml` — core chrome: settings, status bar, toolbar, measure tool,
  object list, Add/Properties windows (`settings.*`, `toolbar.*`, `common.*`,
  `shape.*`, `list.*`, `status.*`, `add.*`, `props.*`).
- `locales/dataset.yml` — the ML dataset visualizer (`dataset.*`).
- `locales/geometry.yml` — the solids/geometry import panel (`geometry.*`).

Add keys to the fragment that matches the area you are touching.

## Coverage

The application UI is fully localized end-to-end: the Settings window (with the
live language picker), toolbar, measurement tool, status bar, object/dataset
list controls, the Add New Object and Object Properties windows, the ML dataset
visualizer and the geometry import panel. Any new string follows the same
`t!("ns.key")` pattern.
